Git上传大致流程

先从总体上了解Git的操作流程，有利于解决错误问题

1
2
3
4
5
6

git init     //在当前的项目目录中生成本地的git管理
git add .  //将文件保存到缓存区,git add -A 将增、删、改的文件改动全保存至缓存区
git commit -m "描述" //将代码从缓存区保存至本地仓库
git remote add origin https://仓库地址.git 
//将本地仓库与指定的远程仓库联系,origin是仓库名字，可以改
git push -u origin master //将本地仓库代码推送至远程仓库

上面就是Git操作大概流程，但是这样看太抽象了，我们用图形来看

git add .

  1. 没有找到.git目录
1
2
3
4

Chun@DESKTOP-6LR6U8B MINGW64 ~/blog (save)
$ git add .
fatal: not a git repository (or any of the parent directories): .git
  //提示表明当前目录没有.git目录

解决：在命令行输入 git init 2. 警告某个目录有.git

1
2

warning: adding embedded git repository: themes/yelee 
//提示在这个目录中有.git

解决：找到themes/yelee 把目录中的**.git** 文件删掉，该文件是隐藏的

git commit

  1. 文件没有被跟踪
1
2
3
4
5
6
7
8

Chun@DESKTOP-6LR6U8B MINGW64 ~/blog (save)
$ git commit -m "first"
On branch save
Changes not staged for commit:
        modified:   themes/icarus (modified content, untracked content)
        modified:   themes/yelee (modified content, untracked content)
#提示icarus与yelee没有被跟踪

解决：删除icarus与yelee文件的.git文件,输入git add -A跟踪所有文件，增，删，改的文件

git remote add

  1. 已经与目的仓库链接
1
2
3

Chun@DESKTOP-6LR6U8B MINGW64 ~/blog (save)
$ git remote add origin https://e.coding.net/Weirdchun/Weirdchun.coding.me.git
fatal: remote origin already exists.

解决：其实可以直接用，但是想换远程仓库的话，就删掉当前仓库，然后再添加

1
2
3
4
5
6

Chun@DESKTOP-6LR6U8B MINGW64 ~/blog (save)
$ git remote -v //查看连接仓库

Chun@DESKTOP-6LR6U8B MINGW64 ~/blog (save)
$ git remote rm origin //删除origin仓库

git push

  1. 远程库与本地库版本不一致 出现错误的主要原因是github中的README.md文件不在本地代码目录中
1
2
3
4
5

Chun@DESKTOP-6LR6U8B MINGW64 ~/blog (save)
$ git push -u coding save
To https://e.coding.net/Weirdchun/Weirdchun.coding.me.git
 ! [rejected]        save -> save (fetch first)
error: failed to push some refs to 'https://e.coding.net/Weirdchun/Weirdchun.coding.me.git'

解决:1. git push -f coding save 忽略版本问题，强制上传并且覆盖(不建议) 2. git pull --rebase origin master(把远程仓库拉到本地同步) 2. 分支没有找到

1
2
3
4

Chun@DESKTOP-6LR6U8B MINGW64 ~/blog (master)  //当前是master分支
$ git push -f coding save     //推送到save分支
error: src refspec save does not match any
error: failed to push some refs to 'https://e.coding.net/Weirdchun/Weirdchun.coding.me.git'

解决： 建立且切换到save分支
